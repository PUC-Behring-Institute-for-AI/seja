# Perspectives

Perspectivas técnicas avaliadas durante o planejamento e implementação.
Cada perspectiva oferece perguntas filtradas por profundidade (essential / full).

## Os 5 Arquétipos do Conselho (D14)

O `seja-council` simula o debate entre 5-7 arquétipos para avaliar decisões
de design conflituosas:

| Arquétipo | Foco | Pergunta central |
|-----------|------|-----------------|
| **Arquiteto** | Integridade estrutural | "Esta decisão compromete a arquitetura a longo prazo?" |
| **Defensor do Usuário** | Experiência e comunicabilidade | "O usuário entende o que está acontecendo?" |
| **Cético** | Riscos e falhas | "O que pode dar errado e como mitigamos?" |
| **Otimista** | Potencial e inovação | "Qual o melhor cenário se investirmos nisto?" |
| **Guardião do Legado** | Consistência e dívida técnica | "Isto aumenta nossa dívida técnica ou paga?" |
| **Economista** | Custo e retorno | "O custo de implementação vale o benefício?" |
| **Pragmático** | Viabilidade imediata | "Dá para fazer com o que temos agora?" |
