# Design Records

Registro de intenção de design (`product-design-as-intended.md`) e estado
construído (`product-design-as-coded.md`).

Cada feature segue o padrão de metacomunicação I/você:
"Eu projetei [X] para você porque [rationale]"

- **Feature ID:** `F-[NNN]` (sequencial por projeto)
- **Status as-coded:** `not_implemented` | `partial` | `implemented`
- **Dualidade:** as-intended (humano mantém) vs as-coded (agente atualiza)
